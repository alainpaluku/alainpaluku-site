## 2025-02-14 - Use optimal loading on images below the fold
**Learning:** In Astro, image loading can be optimized by using `loading="lazy"` and removing `fetchpriority="high"` on images that are below the fold or not critical for LCP (Largest Contentful Paint). On the home page and about page, the profile image is already heavily optimized (inline, mask-image) but they are loaded eagerly with high priority. On the other hand, the `ArticleCard.astro` component has `loading="lazy"` but does not include explicit width/height to avoid layout shift. Also `HomeContent.astro` and `AboutContent.astro` both use the exact same hero image logic. While `loading="eager"` and `fetchpriority="high"` on the profile image in hero is correct for LCP, we can improve performance by explicitly declaring `decoding="async"` on lazy loaded images in `ArticleCard.astro`.
**Action:** Add `decoding="async"` to the `<img>` tag in `src/components/ArticleCard.astro` to allow the browser to decode the image off the main thread.
